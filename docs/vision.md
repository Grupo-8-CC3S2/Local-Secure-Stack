# Visión del Proyecto

Podemos conceptualizar nuestra arquitectura inspirándonos en un escenario típico de AWS. En un sistema de inventario, por ejemplo, los nuevos archivos pueden subirse a un bucket S3, y una función Lambda se activa automáticamente para procesarlos y almacenarlos en una base de datos DynamoDB mediante métodos predefinidos. Este patrón es un ejemplo clásico de arquitectura serverless, en el que la lógica de negocio se desacopla del almacenamiento y se ejecuta bajo demanda.

Nuestra API local sigue un patrón muy similar. Recibe solicitudes de distintos tipos, como chequeos de salud o la creación de recursos, y estas solicitudes son gestionadas por routers de FastAPI, como `APIRouter(prefix="/notes")`. Cada router agrupa endpoints relacionados y actúa como una especie de función Lambda: recibe la solicitud, ejecuta la lógica correspondiente definida en `logica.py` y finalmente persiste los datos en PostgreSQL a través de `data.py` utilizando cursores. Cada nota creada o listada en la API es conceptualmente equivalente a los archivos que se procesan en el bucket S3 y se almacenan en DynamoDB en el escenario AWS.

De manera análoga, en AWS podrían existir funciones Lambda adicionales que verifican ciertas condiciones del inventario, como que la cantidad de un producto no sea cero. En nuestra API, esta función está representada por los endpoints de salud y validación, que aseguran que el sistema funciona correctamente y que los datos se mantienen íntegros. Tras el Sprint 2, esta analogía se hace más evidente: los routers y sus endpoints ejecutan funciones específicas que crean, listan o eliminan recursos, manteniendo la modularidad y escalabilidad de la API.

En conclusión, aunque no estemos usando AWS directamente, la arquitectura de nuestra API refleja los principios de desacoplamiento, modularidad y automatización propios de entornos serverless. Esto permite construir un sistema organizado, mantenible y fácilmente extensible, mientras se reproduce localmente la esencia de un flujo de procesamiento y almacenamiento de datos automatizado y seguro.
